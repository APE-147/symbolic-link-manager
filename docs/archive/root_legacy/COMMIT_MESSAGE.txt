fix: Eliminate TUI display flickering, scrollback pollution, and header residue

## Problem
The TUI suffered from three critical display issues:
1. Screen flickering/jittering during navigation
2. Excessive downward scrolling polluting terminal scrollback
3. Header residue remaining at top after navigation

## Root Causes
- TerminalMenu clear_screen=True causing full screen clear on every redraw
- Multiple unnecessary console.clear() calls
- No alternate screen buffer usage (standard for TUI apps)
- No terminal size adaptation

## Solution
1. Wrapped run_tui() in Rich's console.screen() context manager for alternate screen buffer
2. Set clear_screen=False and clear_menu_on_exit=False in all TerminalMenu instances
3. Added terminal size detection with adaptive preview and menu height limiting
4. Removed unnecessary console.clear() calls, kept only strategic ones for view transitions

## Changes
- Added _get_terminal_size() utility function for terminal dimension detection
- Wrapped TUI main loop in with console.screen() context
- Optimized TerminalMenu parameters: clear_screen=False, clear_menu_on_exit=False
- Added adaptive preview_size (30% on ≥100 cols, disabled on <100)
- Added menu_entries_max_height to prevent overflow
- Removed final console.clear() (context manager handles cleanup)
